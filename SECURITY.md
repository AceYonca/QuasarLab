# Security Policy

QuasarLab is intended for authorized protocol research, education, QA, and defensive lab validation. Use it only on systems, services, and networks that you own or have explicit permission to test.

## Responsible Use

- Keep testing isolated to private lab environments.
- Do not use QuasarLab to access, disrupt, or assess third-party systems without authorization.
- Do not publish live infrastructure details, secrets, private keys, or third-party identifiers in issues or discussions.
- Do not request or contribute persistence, credential access, payload delivery, evasion, or post-exploitation functionality.

## Reporting Issues

If you find a security issue in QuasarLab itself, report it through the repository's private vulnerability reporting flow if available. If private reporting is not enabled, contact the maintainer before posting sensitive details publicly.
